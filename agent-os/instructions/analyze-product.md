name: analyze-product
description: Analyze existing codebase for improvements
---

Analyze an existing codebase to understand its structure and identify improvements.

## 1. Codebase Overview

### 1.1 Project Structure Analysis
- Identify main directories
- Map file organization
- Detect project type (React, Vue, Node, etc.)
- Find entry points
- Locate configuration files

### 1.2 Technology Stack
- Frontend frameworks/libraries
- Backend technologies
- Database systems
- Build tools
- Testing frameworks
- CI/CD setup

## 2. Architecture Analysis

### 2.1 Application Architecture
- Monolith vs Microservices
- Client-server separation
- API design patterns
- State management approach
- Authentication method

### 2.2 Code Architecture
- Design patterns used
- Folder structure logic
- Module organization
- Dependency management
- Cross-cutting concerns

## 3. Feature Inventory

### 3.1 Core Features
Identify and document:
- User authentication
- Main business logic
- Data management
- Admin functions
- Reporting features

### 3.2 Feature Quality
Assess each feature for:
- Completeness
- Test coverage
- Documentation
- Performance
- Security

## 4. Code Quality Assessment

### 4.1 Technical Debt
- Outdated dependencies
- Deprecated APIs
- Code duplication
- Complex functions
- Missing tests

### 4.2 Best Practices
Check adherence to:
- Coding standards
- Security practices
- Performance optimization
- Error handling
- Logging strategy

## 5. Performance Analysis

### 5.1 Frontend Performance
- Bundle sizes
- Load times
- Render performance
- Memory usage
- Network requests

### 5.2 Backend Performance
- API response times
- Database query efficiency
- Caching strategy
- Resource usage
- Scalability concerns

## 6. Security Review

### 6.1 Common Vulnerabilities
- SQL injection risks
- XSS vulnerabilities
- CSRF protection
- Authentication flaws
- Authorization issues

### 6.2 Security Best Practices
- Input validation
- Output encoding
- Secure communication
- Secret management
- Access control

## 7. Improvement Recommendations

### 7.1 Immediate Fixes
- Critical security issues
- Breaking bugs
- Performance bottlenecks
- Missing error handling

### 7.2 Short-term Improvements
- Code refactoring
- Test coverage
- Documentation updates
- Dependency updates

### 7.3 Long-term Enhancements
- Architecture improvements
- Technology migrations
- Feature additions
- Scalability planning

## Generate Analysis Report

Create comprehensive report:
- Executive summary
- Technology overview
- Feature inventory
- Quality metrics
- Security assessment
- Improvement roadmap

Save as `analysis/{project-name}-analysis.md`
